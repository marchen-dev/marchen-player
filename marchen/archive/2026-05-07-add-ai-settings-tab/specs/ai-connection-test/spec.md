## 目的

验证用户配置的 AI 服务商连接是否正常，在保存前给予即时反馈。

### 需求: 测试连接

系统 SHALL 提供测试连接功能，验证 API Key 和 Base URL 的有效性。

#### 场景: 连接成功

- **GIVEN** 用户填写了有效的 API Key 和 Base URL
- **WHEN** 用户点击"测试连接"按钮
- **THEN** 系统发送一个轻量请求验证连通性
- **AND** 显示成功状态（如绿色勾号 + "连接成功"）

#### 场景: 连接失败 - Key 无效

- **GIVEN** 用户填写了无效的 API Key
- **WHEN** 用户点击"测试连接"按钮
- **THEN** 系统显示失败状态和具体错误信息（如 "认证失败: 401"）

#### 场景: 连接失败 - 网络不可达

- **GIVEN** Base URL 指向不可达的地址
- **WHEN** 用户点击"测试连接"按钮
- **THEN** 系统显示失败状态（如 "网络超时"）

#### 场景: 测试中的加载状态

- **GIVEN** 用户点击了"测试连接"
- **WHEN** 请求正在进行中
- **THEN** 按钮显示加载状态，防止重复点击
