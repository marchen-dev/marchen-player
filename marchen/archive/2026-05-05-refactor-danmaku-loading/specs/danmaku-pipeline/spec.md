## 目的

管理从视频导入到弹幕加载完成的完整异步流程，支持暂停（等待用户选择）和取消。

### 需求: 线性加载流程

系统 SHALL 按顺序执行以下步骤：导入视频 → 计算哈希 → 匹配动漫 → 获取弹幕 → 准备播放。每一步完成后 SHALL 更新当前步骤状态。

#### 场景: 正常加载（精准匹配）

- **GIVEN** 用户拖入一个视频文件
- **WHEN** 系统计算哈希并通过 API 精准匹配到动漫
- **THEN** 系统自动获取弹幕（withRelated=true）
- **AND** 弹幕获取成功后保存到历史记录并开始播放

#### 场景: 匹配失败需要用户选择

- **GIVEN** 系统通过哈希未能精准匹配到动漫
- **WHEN** 匹配结果返回 isMatched=false
- **THEN** 系统暂停 pipeline 并弹出匹配对话框
- **AND** 用户选择后 pipeline 从"获取弹幕"步骤恢复

#### 场景: 用户选择跳过弹幕

- **GIVEN** 匹配对话框弹出
- **WHEN** 用户选择"不加载弹幕"
- **THEN** 系统跳过弹幕获取，仅加载本地弹幕（如有），直接开始播放

### 需求: 弹幕缓存策略

系统 SHALL 优先使用 IndexedDB 中的弹幕缓存，仅在缓存不存在或为新番时重新请求。

#### 场景: 使用缓存

- **GIVEN** IndexedDB 中存在该视频的弹幕缓存
- **AND** 该视频不是新番（newBangumi=false）
- **WHEN** pipeline 执行到"获取弹幕"步骤
- **THEN** 系统直接使用缓存数据，不发起网络请求

#### 场景: 新番强制刷新

- **GIVEN** IndexedDB 中存在该视频的弹幕缓存
- **AND** 该视频是新番（newBangumi=true）
- **WHEN** pipeline 执行到"获取弹幕"步骤
- **THEN** 系统忽略缓存，重新请求弹幕

### 需求: 取消加载

系统 SHALL 支持在任意步骤取消加载流程。

#### 场景: 用户导入新视频

- **GIVEN** 当前正在加载一个视频的弹幕
- **WHEN** 用户拖入另一个视频
- **THEN** 系统取消当前 pipeline 并启动新的加载流程

### 需求: 繁简转换

系统 SHALL 通过 API 参数 chConvert 控制繁简转换，不在客户端处理。

#### 场景: 开启繁简转换

- **GIVEN** 用户在设置中开启了"繁体转简体"
- **WHEN** 请求弹幕时
- **THEN** 系统传递 chConvert=1 参数，由服务端完成转换
